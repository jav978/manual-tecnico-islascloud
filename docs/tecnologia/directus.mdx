# Manual de usuario de Directus ![directus](../../static/img/iconos/logodirectus.svg)  [](https://directus.io/ "Página principal de directus")

> El objetivo de esta aplicación directus es administrar la base de datos  de una manera sencilla y eficiente, se podría  decir que directus es una mejor versión de  phpmyadmin, ya que tiene múltiples opciones para gestionar la base de datos.  

> En este documento le explicaremos lo que debe configurar específicamente para realizar la configuraciones y el tratamiento de las colecciones, la permisología de los usuario entre otros. 


## 1. Acceder a la interfaz

Dirígete a la URL donde está alojado Directus e inicia sesión con tus credenciales de usuario. [directus.destraislas.com/](https://directus.destraislas.com/admin/login)

## 2. Explorar la interfaz 

La interfaz de Directus se divide en varios paneles:

- El panel izquierdo muestra las opciones de navegación.
- El panel central muestra el contenido principal.  
- El panel superior muestra la barra de búsqueda y los filtros.


## 3. Gestionar usuarios y permisos

Ve a `Usuarios` en el panel de navegación. Desde ahí puedes:

- Crear nuevos usuarios
- Asignarles roles y permisos

Los permisos controlan a qué paneles y datos puede acceder cada usuario.

## 4. Organizar el contenido en colecciones 

Directus almacena todo el contenido en **colecciones**.

Ve a `Colecciones` en el panel de navegación. Desde ahí puedes crear colecciones para organizar tus datos, como "Artículos", "Proyectos", etc.

## 5. Definir campos de las colecciones

Dentro de cada colección, ve a la pestaña `Campos` para definir los campos que tendrá cada elemento de esa colección.

Puedes crear campos de varios tipos: 

- Texto
- Numéricos
- Booleanos
- Archivos 
- etc.

## 6. Añadir y editar elementos

Dentro de cada colección verás todos sus elementos en una tabla. Usa los botones `+` y `Editar` para:

- Añadir nuevos elementos
- Modificar los existentes

Rellena los campos definidos para cada elemento.
  
## 7. Gestionar archivos y carpetas

En `Archivos` puedes:

- Subir y organizar tus archivos
- Usar carpetas para organizarlos de forma lógica  
- Vincular esos archivos a elementos de tus colecciones

## 8. Crear relaciones entre colecciones

Con `Relaciones` puedes crear vinculaciones entre elementos de diferentes colecciones.

Por ejemplo, puedes vincular:

- Una imagen a un artículo
- Un artículo a una categoría

## 9. Gestionar ajustes

En `Ajustes` encontrarás opciones generales de configuración. 

Por ejemplo, puedes gestionar las colecciones mostradas en el panel de navegación.

---

Eso cubre lo básico para empezar a usar Directus. La interfaz incluye consejos e información contextual para guiarte al usar la plataforma.

¡Prueba a explorar todas las opciones!


## Acerca de Directus ![directus](../../static/img/iconos/logodirectus.svg)  [](https://directus.io/ "Página principal de directus")


La funcion principal de esta aplicación directus es administrar la base de datos  de una manera sencilla y eficiente, se podría  decir que directus es una mejor versión de  **phpmyadmin**, ya que tiene múltiples opciones para gestionar la base de datos.  

En este documento le explicaremos lo que debe configurar específicamente para realizar la configuraciones y el tratamiento de las colecciones, la permisología de los usuario entre otros. 



Esta aplicación será utilizada para el manejo administrativo del Sistema de Reportería Web de Grupo Islas.

Los usuarios que tienen acceso son aquellos que pertenecen al soporte técnico de Grupo Islas, ya que en este podrán realizar configuraciones importantes como:

- **Crear:** organizaciones, usuarios, unidades de medida, grupos de sensores, entre otras configuraciones que permitirán y observará el usuario final.
- Establecer colores.
- Indicar localización exacta de un MCOMM (equipo Flexy).
- Activar o desactivar una organización.
- Realizar la configuración de sensores, a qué grupo pertenece, su nombre largo unidad, entre otros.

Se han creado 12 tablas para esta aplicación:

1. **Alert Info:** Aquí podrá configurar colores de las alarmas y eventos.
2. **Custom Info:** Aquí se indica cuales MCOMM de una organización.
3. **Group Info:** Es la sección para crear y configurar un grupo de sensores.
4. **Mcomm Info:** Aquí se encuentra la información detallada del MCOMM. Es importante asignar correctamente las coordenadas de ubicación para que aparezca correctamente en el mapa del usuario final.
5. **Organización Info:** Aquí deberá indicar el nombre exacto de la organización o empresa.
6. **Owner Info:** Esta es la información del dueño de la aplicación del sistema de reportería.
7. **Sensor Info:** En esta tabla podrá configurar todos los datos del sensor, para que el usuario final pueda ver la información.
8. **Setting Info:** Aquí podrá configurar los estilos del dashboard que está viendo su cliente.
9. **Site Info:** Aquí podrá indicar las coordenadas de la organización.
10. **Type Info:** Aquí se puede crear un listado de la característica que envía el sensor.
11. **Unit Info:** Aquí podrá crear un listado de todas las unidades que desea manejar con su correspondiente símbolo.


# Directus

Directus es una plataforma de administración de contenido de código abierto que funciona con bases de datos SQL y NoSQL. Permite crear un sistema de administración de contenido personalizable. 

## Funcionamiento de Directus

- Directus se instala como una aplicación independiente, típicamente en un servidor Node.js. 

- Provee una API RESTful para acceder y administrar el contenido almacenado en la base de datos.

- La interfaz de administración es una aplicación de JavaScript que se conecta a la API de Directus. Permite operaciones CRUD en los datos.

- Soporta bases de datos como PostgreSQL, MySQL, SQLite y MongoDB para almacenar el contenido.

- Permite definir colecciones personalizadas a través de la interfaz para modelar diferentes tipos de contenido.

- Cada colección puede tener campos de diferentes tipos de datos como texto, imágenes, archivos, relaciones, etc.

- La interfaz de administración permite crear, editar y eliminar registros, así como gestionar las colecciones y campos.

- Tiene funciones de control de acceso y permisos a nivel de usuario y rol.

- Directus puede utilizarse como un headless CMS entregando contenido a través de su API para ser consumido por sitios web y aplicaciones.

## Menú módulo
	
> El menú módulo está agrupado por los siguientes elementos:

- Contenido.
- Directorio del Usuario
- Biblioteca de archivos
- Informes
- Documentación

###  - Página

> Aquí se muestra el contenido de la página actual, incluyendo:

* **Cabecera** - Una sección fija en la parte superior de cada página, la cabecera incluye:

* **Icono de la página** - Al hacer clic en él se vuelve a la página anterior.
* **Nombre del módulo** - Al hacer clic en él, se accede al módulo/sección principal.
* **Título de la página** - Muestra el título dinámico de la página actual.
* **Botones de acción** - Botones contextuales alineados a la derecha para acciones específicas de la página.
* **Contenido de la página** - Muestra el contenido de la página a la que se ha navegado.

### - Barra de módulos

> La sección más a la izquierda de la aplicación es la barra de módulos, que incluye:

> Logotipo del proyecto - Muestra el logotipo y el color del proyecto configurados (por defecto, el logotipo y el color de Directus). Si está configurado, al hacer clic en este componente se navegará a la URL del proyecto. Durante la actividad de la plataforma, también se mostrará aquí un indicador de progreso indeterminado.

> Navegación por módulos - Permite navegar entre los diferentes módulos a los que tiene acceso el usuario. La personalización de la Navegación de Módulos se realiza dentro de la Configuración del Proyecto, pero la lista de módulos por defecto incluye:

* **Contenido** - La forma principal de ver e interactuar con el contenido de la base de datos.
* **Directorio de usuarios** - Una sección dedicada a los usuarios del sistema de la plataforma*
* **Biblioteca de archivos** - Un conjunto de todos los archivos cargados y gestionados dentro de la plataforma
* **Insights** - Acceso a cuadros de mando de datos infinitamente personalizables
* **Guía de la aplicación** - Un portal adaptado, dentro de la aplicación, para los conceptos, guías y referencias de la plataforma
* **Configuración** - Una sección sólo para administradores para configurar el proyecto y los ajustes del sistema
* **Notificaciones** - Abre un cajón de notificaciones, por ejemplo de menciones.
* **Menú de usuario actual** - Este componente muestra el nombre y el avatar del usuario autentificado.
* **Cierre de sesión** - Al pasar por encima del menú de usuario se muestra el botón para cerrar la sesión de la plataforma.

### - Barra de navegación
> La barra de navegación se basa en el módulo actual, e incluye:

* **Nombre del proyecto** - Muestra un icono y un tooltip que indica la fuerza de conexión de la API, y el nombre de su proyecto actual, que puede ser configurado en la Configuración del Proyecto.
* **Navegación** - Se trata de una navegación dinámica basada en su módulo actual. Algunos módulos también admiten preselecciones de marcadores, que son enlaces personalizables a conjuntos de datos específicos.


### - Sidebar
> La barra lateral proporciona contexto y opciones adicionales para la página actual, incluyendo:

* **Componentes de la página** - Enumera cualquier componente contextual de la página disponible. El componente "Información" está disponible en cada página, explicando el propósito general de la página y los detalles relevantes. Al hacer clic en cualquier componente de la barra lateral, se abrirá en acordeón. Si la ventana de la aplicación es lo suficientemente grande, la barra lateral se abrirá automáticamente, y puede cerrarse haciendo clic en la "X" de la parte superior derecha.
* **Bandeja de notificaciones** - Fijado en la parte inferior de la barra lateral de la página, este botón abre una bandeja de notificaciones recientes del usuario, y proporciona un enlace a su página de registro de actividad.

### - Registro de actividad

> El registro de actividad lleva un seguimiento de los eventos y acciones realizadas por los usuarios en el sistema. 

> Cada vez que un usuario interactúa con la plataforma, como crear o editar contenido, iniciar sesión, etc., esa actividad es registrada en el log.

El registro captura información como:

- Fecha y hora
- Usuario
- Tipo de evento (creación, actualización, eliminación, etc)  
- Elemento afectado (colección, item, etc)

Revisar el registro permite:

- Auditar las acciones realizadas
- Solucionar problemas
- Detectar accesos no autorizados

Los administradores pueden filtrar y buscar en el registro para inspeccionar actividades específicas.
